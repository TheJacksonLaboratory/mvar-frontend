Variant calls from reference (GRCm38) based alignment of Illumina short read data (exome and whole genome), and generated from a standard GATK pipeline with modification for laboratory mouse genomes (Fairfield et al., 2015).

Position – the physical position of the variant in the mouse genome reference build GRCm38

Type – the type of variant, SNP: single nucleotide polymorphism, DEL: deletion, INS: insertion

Impact – predicted impact annotation, e.g. from SNPEff, Putative_impact: A simple estimation of putative impact / deleteriousness : (HIGH, MODERATE, LOW, MODIFIER), reference http://snpeff.sourceforge.net/SnpEff_manual.html for details

Annotation – primary annotation described by the Sequence Ontologies, which is a set of terms and relationships used to describe the features and attributes of biological sequence. SO includes different kinds of features which can be located on the sequence. Biological features are those which are defined by their disposition to be involved in a biological process. Examples are binding_site and exon. More details can be found here: http://www.sequenceontology.org/


Structural variant calls from reference (GRCm38) based on whole genome sequence data, and generated from a consensus structural variant calling pipeline based on 4 callers (Breakdancer, Delly, Lumpy, and Manta) for laboratory mouse genomes.

Type – Structural variant type classed into the following:

  • INS – Insertion
  • INV – Inversion
  • DEL – Deletion
  • DUP – Duplication
  • TRA – Translocation

Start Position – Starting position of the SV call. Note for translocation events this position represents a single breakpoint, rather than the start of a range.

End Position – Ending position of the SV call. Note for translocation events this position represents a single breakpoint, rather than the end of a range.

Length – Size of the SV event. Note: For Insertion calls accurate size prediction is not possible with available tools. Rather start and end position show region where an insertion is predicted to be present. For translocations, start and end position represent predicted locations of where a translocation event has occurred, and size prediction is not possible.

Prop. Occurrence - The frequency of the structural variant within the MMRdb variant database.

Supporting Callers – The number of structural variant callers supporting each call. Total possible callers are four.

Mutation Candidate – A variant is a likely pathogenic (y) if its occurrence is low (0-1% frequency where variants found in a single sample are ~0%), its genotype is supported by the phenotype of the sample and the inheritance of the phenotype (e.g. for autosomal recessive phenotypes, an affected individual must be homozygous), and if linkage information is available, the variant must be linked

Sample ID – The ID assigned to each sample when it is added to MMRdb. Use this I.D. to search for additional sample information under ‘Samples’
